# 贡献指南

感谢你考虑为生活经验分享平台做出贡献！

## 如何贡献

### 报告 Bug

如果你发现了 bug，请创建一个 issue 并包含以下信息：

- Bug 的简短描述
- 重现步骤
- 预期行为
- 实际行为
- 截图（如果适用）
- 环境信息（浏览器、操作系统等）

### 提出新功能

我们欢迎新功能建议！请创建一个 issue 描述：

- 功能的目的和用例
- 建议的实现方式
- 可能的替代方案
- 对现有功能的影响

### 提交代码

1. **Fork 仓库**

2. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   # 或
   git checkout -b fix/your-bug-fix
   ```

3. **编写代码**
   - 遵循现有的代码风格
   - 添加必要的注释
   - 确保代码可读性

4. **测试**
   - 测试你的更改
   - 确保不会破坏现有功能

5. **提交**
   ```bash
   git add .
   git commit -m "简短描述你的更改"
   ```

6. **推送**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **创建 Pull Request**
   - 提供清晰的 PR 描述
   - 引用相关的 issue
   - 等待代码审查

## 代码规范

### JavaScript/React

- 使用 ES6+ 语法
- 使用函数式组件和 Hooks
- 组件名使用 PascalCase
- 变量和函数名使用 camelCase
- 常量使用 UPPER_CASE
- 适当添加注释

### 文件命名

- React 组件：`ComponentName.jsx`
- 样式文件：`ComponentName.css`
- 工具函数：`utilityName.js`
- 常量：`constants.js`

### Git 提交信息

使用清晰的提交信息：

- `feat: 添加新功能`
- `fix: 修复 bug`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 代码重构`
- `test: 添加测试`
- `chore: 构建/工具更改`

## 开发流程

1. 在本地设置开发环境（参考 README.md）
2. 创建新分支进行开发
3. 进行开发和测试
4. 提交代码并创建 PR
5. 等待代码审查
6. 根据反馈进行调整
7. 合并到主分支
